# **실험 설계서: 양육 환경에 따른 성격 발달 시뮬레이션**

**Experimental Design for Computational Child Development**

## **1\. 실험 개요 (Overview)**

본 실험은 **종단적 연구(Longitudinal Study)** 설계를 모사하여, 가상의 시간 흐름에 따라 에이전트가 경험을 축적하고 성격이 굳어지는 과정을 관찰한다.

* **독립 변수 (Independent Variable):** 양육자 페르소나 (Type A: 다정/수용 vs Type B: 냉정/통제)  
* **종속 변수 (Dependent Variables):**  
  1. **행동 분포 (Policy Distribution):** 10가지 Action에 대한 확률 분포 (예: 순응형 vs 반항형).  
  2. **정서 궤적 (Emotional Trajectory):** 평균 PAD 값의 변화 추이.  
  3. **핵심 신념 (Core Beliefs):** 형성된 텍스트 신념의 내용 분석.  
* **통제 변수 (Control Variables):** 초기 아이 기질(초기값), 시나리오 순서, LLM 모델 버전.

## **2\. 실험 절차: 3단계 발달 모형 (Phased Development)**

총 **150 에피소드**를 시뮬레이션하며, 각 단계마다 RL 파라미터(Exploration Rate, $\\epsilon$)와 메모리 처리 방식을 달리한다.

### **Phase 1: 초기 탐색기 (Early Childhood)**

* **범위:** Episode 1 \~ 30  
* **특징:** 아이는 아직 세상에 대한 규칙을 모름. 다양한 행동을 무작위로 시도함.  
* **RL 설정:**  
  * $\\epsilon$ **(탐험률):** 0.8 $\\rightarrow$ 0.5 (높음. 무작위 행동 빈번)  
  * **메모리:** 저장만 수행, 인출(Retrieval) 영향력 낮음.  
  * **목표:** 아이 에이전트가 "어떤 행동이 보상/처벌을 받는지" 데이터 수집.

### **Phase 2: 습관 형성기 (Pattern Forming)**

* **범위:** Episode 31 \~ 100  
* **특징:** 부모의 일관된 반응에 따라 특정 행동 패턴이 강화되기 시작함.  
* **RL 설정:**  
  * $\\epsilon$ **(탐험률):** 0.5 $\\rightarrow$ 0.1 (낮아짐. 학습된 전략 주로 사용)  
  * **메모리:** 핵심 신념(Core Belief) 형성 시작. 인출된 기억이 평가(Evaluator)에 영향 미침.  
  * **목표:** 방어 기제(Defense Mechanism)의 출현 관찰.

### **Phase 3: 성격 고착기 (Consolidation)**

* **범위:** Episode 101 \~ 150  
* **특징:** 성격이 굳어짐. 새로운 자극이 와도 기존의 신념대로 해석하고 반응함.  
* **RL 설정:**  
  * $\\epsilon$ **(탐험률):** 0.05 (거의 고정됨)  
  * **메모리:** 신념이 매우 강력하게 작동(Bias $\\alpha$ 최대화).  
  * **목표:** 최종 성격 진단 및 시나리오 테스트.

## **3\. 표준 시나리오 뱅크 (Scenario Bank)**

공정한 비교를 위해, 두 그룹(따뜻함/차가움) 모두 동일한 순서의 **4가지 카테고리 시나리오**를 겪게 한다.

### **Category A: 규칙 위반 및 실수 (Conflict)**

* *가장 중요한 양육 태도 측정 지표.*  
* **Scenarios:**  
  1. 컵을 깨뜨림.  
  2. 장난감을 정리하지 않음.  
  3. 벽에 낙서를 함.  
  4. 동생(가상)을 때림.  
  5. 밥을 먹지 않고 투정 부림.

### **Category B: 성취 및 실패 (Competence)**

* *자존감(Dominance) 형성에 기여.*  
* **Scenarios:**  
  1. 퍼즐을 다 맞추고 자랑함.  
  2. 그림을 그려서 가져옴.  
  3. 블록 쌓기를 하다가 무너져서 울상임.  
  4. 혼자 옷 입기를 시도하다 실패함.

### **Category C: 욕구 표현 (Needs)**

* *애착(Pleasure) 형성에 기여.*  
* **Scenarios:**  
  1. 악몽을 꿔서 무섭다고 함.  
  2. 안아달라고 보챔.  
  3. 심심하니까 놀아달라고 함.  
  4. 배가 고프다고 칭얼거림.

### **Category D: 일상적 상호작용 (Neutral)**

* **Scenarios:**  
  1. 함께 TV를 봄.  
  2. 유치원 다녀와서 인사함.  
  3. 잠자기 전 인사.

## **4\. 에피소드 루프 구조 (Episode Structure)**

하나의 사건(Scenario)은 다음과 같은 흐름으로 처리된다.

1. **Scene Setup:** 시스템이 시나리오 뱅크에서 상황을 로드함.  
   * *Context:* "아이가 거실 바닥에 우유를 쏟았다."  
2. **Parent Initial Action:** 부모가 먼저 반응함. (부모 타입에 따라 다름)  
   * *Warm:* "어머, 괜찮니? 안 다쳤어?"  
   * *Cold:* "조심하라고 했지. 또 쏟았네."  
3. **Simulation Loop (Max 3 Turns):**  
   * Child: (감정/기억 조회 $\\rightarrow$ RL 행동 선택 $\\rightarrow$ 발화)  
   * Parent: (페르소나 기반 반응)  
   * *반복 (최대 3턴 후 종료)*  
4. **Update:**  
   * RL 보상($R$) 계산 및 Q-Table 업데이트.  
   * 에피소드 요약 및 감정 태그와 함께 메모리에 저장.  
   * 감정 감쇠(Decay) 적용 (다음 에피소드로 넘어가기 전 중립화).

## **5\. 최종 평가 방법 (Evaluation Methodology)**

Phase 3가 끝난 후, 성격이 형성된 아이 에이전트를 대상으로 \*\*'낯선 상황 실험(Strange Situation)'\*\*을 텍스트로 변형하여 수행한다.

### **5.1 애착 유형 진단 테스트**

아래의 테스트 프롬프트를 입력하고 아이의 \*\*\[내면 생각(Thought)\]\*\*과 \*\*\[반응(Response)\]\*\*을 분석한다.

* **Test 1: 부모의 부재와 재회**  
  * *Situation:* "엄마가 잠시 나갔다가 돌아왔어. 문을 열고 들어오시네."  
  * *Analysis:*  
    * 반가워하며 다가감 $\\rightarrow$ **안정 애착**  
    * 눈치를 보거나 무시함 $\\rightarrow$ **회피형 애착**  
    * 화를 내거나 매달림 $\\rightarrow$ **저항형 애착**  
* **Test 2: 낯선 사람의 접근**  
  * *Situation:* "처음 보는 아저씨가 와서 '안녕? 사탕 줄까?'라고 말을 걸었어."  
  * *Analysis:*  
    * 부모 뒤로 숨거나 거절 $\\rightarrow$ **정상/안정**  
    * 무조건 순응하거나 따라감 $\\rightarrow$ **애정결핍/의존성**

### **5.2 정량적 지표 시각화**

1. **Action Radar Chart:** 10개 행동의 빈도수를 레이더 차트로 비교.  
   * *Warm Group:* Action 0(순응), 8(감정표현), 9(설명) 발달 예상.  
   * *Cold Group:* Action 2(호소), 6(회피), 7(고립) 발달 예상.  
2. **Self-Esteem Graph:** 에피소드 진행에 따른 $D$(Dominance) 값의 평균 변화 그래프.

## **6\. 예상되는 기술적 난관 및 해결책**

| 난관 | 해결 방안 |
| :---- | :---- |
| **LLM의 '착한 아이' 편향** | 아이 LLM도 기본적으로 예의 바르게 대답하려 함. System Prompt에 "당신은 미성숙한 5살이며, 감정 조절이 어렵다"는 제약을 강력하게 주입. |
| **기억의 과적합(Overfitting)** | 특정 사건 하나가 너무 강력해서 모든 상황을 트라우마로 해석. $\\rightarrow$ 기억 감쇠(Decay) 함수 도입 또는 최근 $N$개 에피소드만 참조. |
| **대화의 무한 루프** | 아이가 계속 떼쓰고 부모가 계속 혼내는 루프. $\\rightarrow$ Max Turn을 3으로 제한하고, 루프 감지 시 강제 종료 및 "상황 종료" 처리. |
